Local MLX Model Lifecycle

Wireframe for the model-library flow inside Settings. It shows the happy path for curated installs, the custom Hugging Face URL/ID entry, and the managed-delete confirmation. The UI stays explicit about what Odyssey owns and what it will never delete.

Download Flow

1
Choose a source Pick a top MLX model card or paste a Hugging Face repo URL/ID.
2
Normalize before install Show the canonical repo id Odyssey will use, such as mlx-community/Qwen2.5-1.5B-Instruct-4bit.
3
Install with ownership metadata Save the repo id, source URL when present, and managed path so deletion is deterministic later.
4
Refresh the installed list Offer a one-click “Set as Default” action once the download completes.

Delete Flow

1
Delete only managed installs The delete button appears for managed MLX downloads only, never for user-supplied local paths.
2
Confirm what will be removed Show the repo id and managed cache path that Odyssey recorded at install time.
3
Remove manifest entry and managed files Delete the tracked model path, prune empty folders, and refresh readiness state in Settings.
