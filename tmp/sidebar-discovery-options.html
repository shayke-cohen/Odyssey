Sidebar Discovery Review

`New` got clearer, but agent and group discovery got quieter.

The sidebar now has a strong creation CTA, but it no longer exposes “how do I start with an agent or a group?” as a first-order decision. The fix is not to remove the simpler sidebar, but to bring back one visible path to people and teams without returning to toolbar clutter.

Problem `New Thread` and `Group Thread` are available, but they hide the mental model: agents and groups are reusable assets you can start from.
Goal Keep the simplified chrome, but make agent/group discovery visible within one glance of the sidebar.
My Lean Option A is the safest improvement. Option B is strongest if you want more “start from capability” behavior.

Option A

Recommended
Keep the big `New` CTA, but add a visible `Agents` and `Groups` discovery strip directly under it. This restores discoverability fast, with minimal structural change.

What changes

You still get one primary `New` button, but the next shelf quietly advertises the agent and group system.

Why it helps

It makes reusable assets feel alive through recent examples instead of extra buttons.

Tradeoff

Discovery is softer. It helps returning users most, but teaches new users less directly than Option A or B.

Recommendation

  • Choose Option A if you want the least risky fix: keep the current sidebar pattern, but restore visible discovery for agents and groups.
  • Choose Option B if you want the product model to feel more intentional: users start by choosing the kind of conversation they want.
  • Choose Option C if you want to preserve the current simplicity almost entirely and just add better signposting.
My strongest recommendation is Option A: it solves the discovery problem without undoing the calmer chrome work we just did.
