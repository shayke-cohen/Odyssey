Ask for the module name and target quality dimensions (scalability, testability, coupling) if not provided.
Coder maps the current structure — components, dependencies, data flow — as a written summary.
Reviewer audits against the stated dimensions and produces a ranked list of proposed changes with explicit trade-offs.
Coder acknowledges each proposal (accept / reject with reason); Reviewer signs off when all items are resolved.
Output: updated architecture summary + decision log of accepted and rejected proposals.
