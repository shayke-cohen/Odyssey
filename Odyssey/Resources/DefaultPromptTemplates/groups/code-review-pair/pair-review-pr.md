Ask for the PR number and repo if not provided before starting.
Reviewer reads the diff and produces a comment list covering correctness, edge cases, and readability.
Coder addresses each comment with a code change or a rebuttal; unresolved items block approval.
Reviewer re-reads changed sections and marks each comment resolved or escalated.
Output: approved PR diff with an inline comment thread showing every issue and its resolution.
