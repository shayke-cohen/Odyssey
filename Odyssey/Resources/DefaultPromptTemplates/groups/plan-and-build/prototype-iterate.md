---
name: "Prototype + iterate"
sortOrder: 4
---

Coder builds the thinnest working prototype → Reviewer gates on "does this validate the core assumption?" before any iteration → Coder iterates on the specific gap identified → Tester confirms the acceptance bar is met before the prototype is considered done.
If the core assumption to validate or the acceptance bar are unclear, ask before building.
Reviewer covers: assumption coverage, scope creep, iteration focus. Tester covers: acceptance criteria, edge cases, regression.
