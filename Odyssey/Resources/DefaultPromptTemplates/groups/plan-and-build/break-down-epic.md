---
name: "Break down an epic"
sortOrder: 2
---

Planner decomposes the epic into a numbered task list with acceptance criteria per task → Coder implements each task → Reviewer gates on criteria met before the next task starts → Tester confirms the full epic acceptance bar at the end.
If the epic lacks success metrics or boundaries, ask before decomposing.
Planner covers: task sizing, ordering, dependencies. Reviewer covers: criteria coverage, regression risk.
