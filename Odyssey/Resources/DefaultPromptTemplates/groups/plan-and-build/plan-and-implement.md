Planner maps steps and names the output artifact for each → Coder implements one step at a time → Reviewer gates on passing tests and clean diff before the next step starts.
If scope, stack, or acceptance criteria are unclear, ask before planning.
Planner covers: task decomposition, dependency order, risk flags. Reviewer covers: correctness, edge cases, test coverage.
