Page:

If the page URL or current conversion rate is unknown, ask before starting.
Analyst audits funnel data — traffic source, drop-off points, scroll depth — and produces a `conversion-audit.md` naming the top friction.
Writer rewrites headline and CTA copy targeting the top friction; Designer proposes layout changes — both must reference `conversion-audit.md`.
Engineer implements the changes behind an A/B flag; Analyst confirms tracking is live before the test goes live.
Gate: audit must exist and changes must be traceable to a specific finding before shipping.
