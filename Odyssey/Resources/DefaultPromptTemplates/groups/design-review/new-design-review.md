Ask for the design file or description and the user goal it serves if not provided.
UX Reviewer evaluates task flow, information hierarchy, and error states; output: ranked list of UX issues.
Engineer assesses technical feasibility — platform constraints, animation cost, data dependencies.
Pattern Reviewer flags deviations from established design-system components and tokens.
Designer resolves all blocking issues; all three reviewers approve. Output: review report with issues, verdicts, and required changes per role.
