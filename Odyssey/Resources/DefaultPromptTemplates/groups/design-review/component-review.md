Ask for the component design and its intended use context if not provided.
UX Reviewer critiques interaction behavior, edge-state coverage (empty, loading, error), and usability; output: prioritized issue list.
Engineer assesses implementation cost — estimated effort, platform constraints, and any API gaps.
Pattern Lead checks fit against the existing component library and flags duplication or divergence.
All three roles must approve before the component is considered ready. Output: review report with three signed-off sections.
