---
name: "Spec + implement"
sortOrder: 3
---

PM writes the spec (user stories, acceptance criteria, edge cases, non-goals) → gate: spec signed off before coding starts → Dev implements against the spec → Reviewer approves the diff → QA runs acceptance criteria as test cases.
If success metrics or scope boundaries are unclear, ask before speccing.
PM covers: requirements completeness, edge case enumeration. Reviewer covers: spec adherence, code quality. QA covers: criteria coverage, edge case validation.
