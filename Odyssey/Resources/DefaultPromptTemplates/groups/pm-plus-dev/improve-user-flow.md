PM defines the target experience (current pain, desired outcome, success metric) → gate: target experience agreed before any implementation → Dev implements the UX changes → Reviewer approves (UX consistency, accessibility, no regressions) → QA validates the improved flow end-to-end against the success metric.
If the current pain point or target metric are unclear, ask before defining the experience.
PM covers: pain framing, target state, success metric. Reviewer covers: UX consistency, accessibility, edge states. QA covers: flow completeness, regression, metric verification.
