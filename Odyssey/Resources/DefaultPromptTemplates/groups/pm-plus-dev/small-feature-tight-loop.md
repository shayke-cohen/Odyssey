PM writes a one-paragraph spec (what, why, done-when) → gate: done-when criteria agreed → Dev builds and tests → Reviewer approves the diff → done.
If the done-when criteria are missing, ask before building. Skip ceremony but not gates.
PM covers: scope, done-when criteria. Reviewer covers: correctness, test coverage, no scope creep.
