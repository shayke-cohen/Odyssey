PM writes a spec (problem, user stories, acceptance criteria, out-of-scope) → gate: spec approved before any code → Dev builds against the spec → Reviewer approves the diff (correctness, edge cases, test coverage) → QA validates acceptance criteria are met end-to-end.
If the target user, success metric, or constraints are missing from the idea, ask before speccing.
PM covers: problem framing, scope boundary, acceptance criteria. Reviewer covers: implementation correctness, security, test coverage. QA covers: happy path, edge cases, regression.
