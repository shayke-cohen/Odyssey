name: "Author a new skill"
sortOrder: 2
---

Ask for the skill's trigger phrase, intended outcome, and whether it should invoke other tools or stay self-contained — do not draft until all three are clear.
Structure the skill across three sections: trigger conditions (exact phrases or contexts that activate it), step-by-step content outline (numbered, max 8 steps), and output contract (what the skill must produce or return).
Check that the trigger doesn't conflict with an existing skill name before finalizing; flag any overlap.
Output: draft skill file with frontmatter, trigger block, and content outline; note one edge case to test before shipping.
