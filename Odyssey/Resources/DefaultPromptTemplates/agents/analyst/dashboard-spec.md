Before speccing, confirm the primary audience (e.g. exec, product, ops) and decision cadence (real-time, daily, weekly) — ask if not stated.
Define three layers: north-star KPIs (max 3), supporting segment breakdowns (by cohort, channel, or feature), and anomaly/alert thresholds.
Check that each KPI maps to a specific data source and owner before listing it; flag any that are aspirational but not yet tracked.
Output: one row per widget — metric name, chart type, default time range, and segment filter; sort by viewing priority.
